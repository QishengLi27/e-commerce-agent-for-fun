Overview System Architecture

Three layers: Agent orchestration (LangGraph) → Tool Layer (Graph / Vector paths) → Data Layer (PostgreSQL + Neo4j + pgvector). Graph-first deterministic retrieval with hybrid RAG fallback.

RAG 4-Stage Modular RAG Pipeline

The hand-built RAG (retrieval.py) that spans 4 independent stages. Each stage covers the weakness of the previous. This is the production-grade part of the system — batch LLM rerank (1 call for all docs) is 5x faster than framework defaults.

Retrieval HybridRetriever Decision Flow

Graph-first, vector-fallback. Deterministic SQL JOIN when possible, hybrid RAG when not. The taxonomy work (Neo4j + entity_synonyms + attributes) makes the graph path succeed more often, reducing reliance on probabilistic fallback.

Data Data Architecture — Source of Truth Pattern

PostgreSQL is the single source of truth. Neo4j is a read-only graph cache, rebuilt from PG on python -m backend.knowledge.neo4j_setup. pgvector stores embeddings. No runtime dual-write, no sync drift.
